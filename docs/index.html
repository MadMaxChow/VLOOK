接口验证码生成规则

  1. 生成接口验证码 token 涉及的参数，不同接口均不同，具体请查阅对应接口说明中的 token 字段中的说明；

  2. 所有参与生成接口验证码的参数名，作为 key 并以降序排列；

  3. 参数对应的值 value 以字符串形式串接，并将 AppSecret 拼接到字符串末尾，最后以 MD5 方式进行加密（32 位，位数不够前面补 0）生成对应的 token 。

生成接口验证码 (token) 的示例

如某接口中生成验证码涉及以下字段：

1
// 字段名 key : 字段值 value
2
"ts" : 1646985087032
3
"channel" : "SHQC"
4
"type" : "unionId"
5
"value" : "oFGdtwJwZvbbhzIVYFW7b1tOmDI4"

  1. 以 key 按 desc 方式排序，按顺序读出每个字段的 value 进行字符串拼接，最后再加上 AppSecret 后的内容为： 1646985087032oFGdtwJwZvbbhzIVYFW7b1tOmDI4unionIdSHQCae929b16ddc5144e1068cdd89e0ca439 ；

  2. 以 MD5 方式加密字符串，得到该接口的验证码 3e7b924aa761ef079f7305f219c1dbdd 。

  • 完整的企微好友数据库是每天增量更新；
