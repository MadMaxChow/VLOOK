Markdown 是什么？

  • 2004 年，John Gruber 创造了 Markdown，一种专门针对网络写作的 文本标记语言 。使用 Markdown，你只需在写作过程中插入少量的标记符号，就能很轻松地进行排版（例如设置标题、加粗、列表、引用块等）；

  • Markdown 文档以 纯文本格式存储 ，这意味着，它们可以用几乎任一种文本编辑器打开。同时，又能通过 Markdown 编辑器导出为带排版的富文本文档、HTML 网页等等。纯粹、简洁、易用、灵活，都是人们喜欢 Markdown 的原因。

🖥️ 离线模式

将主题和插件一并打包到生成的 HTML 中，适合没有自己的 Web 站点，或不依赖网络直接打开阅读的情况。

配置导出选项

tab

导出为 HTML ❯

  1. 创建导出 HTML 配置：

    1. 打开 Typora 导出配置

    2. 打开菜单 Typora > 文件 > 导出 > 导出配置

    3. 添加（选择 HTML 模板），并将配置命名为 VLOOK-HTML

  2. 安装插件：

    1. 清空 在 <head /> 中添加 配置的内容

    2. 打开插件文件：released/plugin/plugin.txt

    3. 全选，并复制所有内容

    4. 将复制的内容粘贴至配置项内

导出为 PDF ❯

  1. 创建导出 PDF 配置：

    1. 打开 Typora 导出配置

    2. 打开菜单 Typora > 文件 > 导出 > 导出配置

    3. 添加（选择 PDF (Typora / WebKit) 模板），并将配置命名为 VLOOK-PDF

  2. 安装插件（可选，针对应用了 VLOOK™ 扩展语法的文档）：

    1. 清空 插入额外内容 (HTML) 配置项内容

    2. 打开插件文件：released/plugin/plugin.txt

    3. 全选，并复制所有内容

    4. 将复制的内容粘贴至配置项内

示例文件

  1. 打开符合 VLOOK™ 规范的 md 文件（参考 released/samples 下的文件 ）

  2. 点击菜单 Typora ▸ 文件 ▸ 导出 ，选择以上创建的导出配置进行导出

Tip

当前文档，以及 VLOOK™ 使用说明攻略的所有 Markdown 文档及更多示例文档，详见 released/samples 下的文件。

 

Warning

目前支持两种导出 PDF 的方式：

  • 方式一：Typora 直接导出普通 PDF 不涉及排版增强

  • 方式二：Typora 直接导出增强 PDF 涉及排版增强，须安装插件

  • 方式三：先导出 HTML 再发布为 PDF 适用于控制更多导出 PDF 的选项。 即先通过以上介绍的「导出为 HTML」并用浏览器打开，再使用 VLOOK™ 在提供的「发布为 PDF」功能（位于页面的右下方），可提供更多的导出选项，详见「这里」

🚨 注意 🚨

在使用「方式一、二」时如有以下情况，建议先使用「方式三」导出 PDF：

  • 希望对导出的 PDF 的内容有更多的控制调整

  • Windows 环境：使用的 Typora 版本为 1.12.x 或更老的版本

  • macOS 环境：文档中有 Mermaid 图表，或主题有渐变半透明 / 渐变的元素

（以上部分问题将会在 Typora 的 1.13.x 版本后会修复）

在线字体风格

tab

部署网络字体

  1. 在 Web 站点新建的 VLOOK 「字体风格资源目录」（如：fonts ）

  2. 在 GitHub 上下载 openfonts 的最新版本（如：web-font-V2.0.tar.gz ）

  3. 解压后将里面的所有子目录（如：s v ）上传至 Web 站点的「字体风格资源目录」下（如：fonts）

引用网络字体

  1. 用文本编辑器打开 released/theme-live/V20xx.x 下的所有以 fs- 开头的 CSS 文件

  2. 搜索所有 <your-host> 的内容，替换为 Web 站点的域名，并保存

  3. 完成更新后的文件上传更新至 Web 站点的「主题资源目录」下（如：V2025.10）

 

Important

  • 若站点上不是按建议的名称创建不同的资源目录，如：fonts 、VLOOK 的版本号等，在更新 <your-host> 外，还要更新其后对应的资源目录名称；

  • 若使用旧版本的主题、插件所发布的 HTML 不重新发布，则需要保留原来版本的目录，不要删除

导出选项

tab

配置引用资源

  1. 用文本编辑器打开 released/plugin/plugin-live.txt

  2. 搜索所有 <your-host> 的内容，替换为 Web 站点的域名，并保存

  3. 涉及的配置项及说明如下：

    • <link rel="preconnect" ...> — 预连接的服务器域名

    • <link rel="dns-prefetch" ...> — （同上）

    • <meta name="vlook-js" ...> — js 文件所在子目录的 URL

    • <meta name="vlook-fs" ...> — 网络字体文件所在子目录的 URL
